# Cloudflare Pages Deployment Guide

## Quick Setup

### 1. Deploy PartyKit Backend (First!)

```bash
npm run party:deploy
```

This will deploy your multiplayer backend and give you a URL like:
`https://opencatan.partykit.dev`

**Save this URL** - you'll need it in step 3.

### 2. Create Cloudflare Pages Project

1. Go to [Cloudflare Dashboard](https://dash.cloudflare.com/)
2. Click **Workers & Pages** → **Create application** → **Pages**
3. Click **Connect to Git**
4. Select your GitHub account and **OpenCatan** repository
5. Click **Begin setup**

### 3. Configure Build Settings

Use these settings:

- **Production branch**: `master`
- **Build command**: `npm run build`
- **Build output directory**: `dist`
- **Root directory**: `/` (leave blank)

**Environment Variables** (click "Add variable"):
- `NODE_VERSION`: `18` or `20`
- `VITE_PARTYKIT_HOST`: `your-project.your-username.partykit.dev` (get from step 1)

### 4. Deploy!

Click **Save and Deploy**. Cloudflare Pages will build and deploy your site.

Your site will be live at a URL like: `https://opencatan-xxx.pages.dev`

### 5. Save Your URLs

- **Cloudflare Pages**: `https://opencatan.pages.dev` (or similar)
- **PartyKit**: `https://opencatan.partykit.dev` (from step 1)

## Automatic Deployments

Every push to `master` will automatically trigger a new deployment on Cloudflare Pages!

## Preview Deployments

Every pull request gets its own preview URL automatically.

## Custom Domain (Later)

Once ready, you can add your custom domain in:
Cloudflare Pages → Your Project → Custom domains
